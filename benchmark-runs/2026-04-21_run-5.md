### 2. System Prompt: Complete Rewrite

Rewrote the entire `systemConstraints()` prompt based on analysis of top-scoring agents
(Apex2 at 64.5%, Claude Mythos at 82%, Warp at 52%). Key improvements:

**Structural changes:**
- Renamed "Core Rules" to "Absolute Rules" with numbered, imperative rules
- Changed workflow from "Observe → Plan → Implement → Verify → Commit" to
  "Observe → Implement → Commit → Verify → Iterate" — commit BEFORE verify
- Reduced 5 phases to 5 steps with tighter guidance

**Observation improvements:**
- Replaced two-command observation with a single comprehensive `find && cat && which` command
- Added checks for more build systems: pyproject.toml, setup.cfg, build.gradle, pom.xml
- Added compiler version checks in the observation command
- Strengthened "do NOT exhaustively explore" directive

**Implementation improvements:**
- Added Python-based file writing as fallback for heredoc issues
- Added mandatory read-back verification after every file write
- Added "delete and rewrite from scratch" when file content is wrong
- Added `mkdir -p` reminder for tasks needing specific paths
- Added combined pip install fallback chain in single command
- Added nohup guidance with verification (curl/nc)

**Commit discipline (biggest change):**
- Moved commit BEFORE verification — "a committed solution scores points even if
  verification reveals issues"
- Added troubleshooting for "nothing to commit" (wrong directory check)
- First commit now expected within budget/6 seconds (was budget/3)

**Verification improvements:**
- Added `diff <(command) <(echo 'expected')` for precise output comparison
- Added explicit checklist: trailing newlines, whitespace, numeric precision,
  case sensitivity, file encoding
- Added combined exit code checking pattern

**Error recovery overhaul:**
- Replaced flat list with 4-level escalation: specific fix → different approach →
  simplify drastically → different language/tools
- More concise, actionable common fixes section

**Time management improvements:**
- Tighter time budget phases: first commit at budget/6 (was budget/3), stop improving
  at 65% (was 70%), final commit at 85% (was implied)
- Reduced max wait per command from 60s to 45s
- Added explicit "commit partial results if timeout kills process"
- Added emphasis: "uncommitted perfect solution scores ZERO"

## Analysis: Why These Changes Should Improve Scores

Based on studying the leaderboard, the key differentiators between agents scoring 50%+ vs
agents scoring <20% are:

1. **Commit discipline** — top agents commit early and often. Our previous prompt said
   "commit within 2-3 minutes" which is too late for simple tasks. Now it's budget/6 seconds
   (~140s for 840s budget), and critically, commit happens BEFORE verification.

2. **Output precision** — many task failures come from wrong whitespace, wrong newlines,
   wrong numeric formatting. Added explicit diff-based comparison and checklist.

3. **Error recovery speed** — top agents don't waste time retrying the same approach.
   The 4-level escalation forces the agent to change strategy after just one retry.

4. **Model capability** — Claude Opus 4.7 has measurably better agentic coding. The model
   upgrade alone should improve scores on tasks where 4.6 was close but not quite right.

5. **Environment readiness** — pre-installing Rust/Cargo eliminates a common failure mode
   where Rust-based tasks fail due to missing toolchain.

## Strategy Note

All improvements are general-purpose patterns, not task-specific hacks:
- Earlier commits and commit-before-verify is universally beneficial
- Better file writing hygiene (verify after write) prevents a whole class of failures
- The error escalation ladder helps on any task, not just specific ones
- Rust toolchain is a general capability, not a task-specific dependency

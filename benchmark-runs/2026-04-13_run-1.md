## Failure Mode: AgentTimeoutError

Harbor killed the agent after **900 seconds** without any committed output.

### What happened
- Claude initialized and explored the checkpoint format (8 Bash tool calls, ~206KB of log)
- Then entered extended thinking for the C code generation
- Never produced a file or commit before the 900s deadline

### Root cause
gitgang had no time-budget awareness. Claude was never told it had a deadline,
and had no incentive to commit a partial solution early. Extended thinking on
a hard code-golf problem consumed the entire budget.

### Fix applied (commit 7ee1b14)
- `systemConstraints()` now injects time budget + "commit minimal solution first"
  strategy into the agent prompt when `GITGANG_TIME_BUDGET_SECONDS` is set
- `runClaude()` wraps the subprocess with `timeout N` for graceful exit
- Harbor agent passes `GITGANG_TIME_BUDGET_SECONDS=840` (900s - 60s buffer)

This is a general improvement for any time-constrained execution context.

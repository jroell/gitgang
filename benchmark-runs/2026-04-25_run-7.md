## Changes Applied (This Run)

### 1. Reasoning Effort: xhigh → high for Benchmark Mode (Major)

**Research basis:** LangChain's harness engineering blog showed that running Claude at
xhigh-only scored 53.9% due to agent timeouts on Terminal-Bench 2.0, while running at
high scored 63.6%. Their optimal "reasoning sandwich" (xhigh-high-xhigh) scored 66.5%.

Claude Code doesn't support mid-session effort switching, so we can't do a true sandwich.
Instead, we now use `--effort high` in benchmark mode (time-budgeted runs) and keep
`--effort xhigh` for interactive use. This is the single highest-impact change based
on available research — avoiding timeouts is worth more than deeper reasoning per turn.

**Implementation:**
- `runClaude()` now checks `isBenchmark` and sets effort accordingly
- Reviewer agent stays at xhigh (short-lived synthesis task, not affected by timeouts)

### 2. System Prompt: Self-Verification Emphasis + Loop Detection

**Research basis:** LangChain's research showed that concentrating reasoning compute on
verification phases (the "xhigh" bookend in their sandwich) provided the biggest score
improvement. Since we can't change effort mid-session, we compensate by making the
prompt strongly emphasize verification.

Additionally, LangChain's LoopDetectionMiddleware (which injects "reconsider your
approach" after N edits to the same file) improved their score. We implement this as
prompt-level self-monitoring guidance.

**Key changes:**
- Added "## Step 4: Test and verify (invest time here)" with emphasis that this is
  "the most important step"
- Added "## Loop Detection (self-check)" section: if you've edited the same file 3+
  times and tests still fail, stop and reconsider fundamentally
- Added rule: "Your work will be measured by automated programmatic tests"
- Added guidance to check edge cases and verify against ALL test examples
- Added "If CLAUDE.md exists with environment context, use it" to avoid redundant discovery
- Strengthened "Before finishing" with "word by word" re-reading emphasis

### 3. Time Management: Less Aggressive Deadlines

**Research basis:** The budget/8 first commit deadline from run-6 (105s for 840s budget)
was likely too aggressive for complex tasks (ML, security, multi-file projects). Research
shows simple tasks complete quickly regardless, while complex tasks need the extra time.

**Changes:**
- First commit deadline: budget/6 (~140s, was budget/8 ~105s) — more breathing room
- Feature freeze: 65% of budget (was 60%) — more time for iteration
- Final commit: 85% of budget (was 80%) — less wasted buffer
- Command timeout: 45s (was 30s) — some builds/tests legitimately need >30s

### 4. Harbor Agent: Pre-install Common Python Packages

**Research basis:** Terminal-Bench research shows 24.1% of failures are from missing
tools/executables. Pre-installing common Python packages eliminates mid-task `pip install`
delays and potential failures.

**Packages added:** numpy, scipy, pandas, matplotlib, requests, flask, pytest,
pyyaml, toml, jsonschema, cryptography, pycryptodome, beautifulsoup4, lxml, pillow,
sympy, networkx

### 5. Harbor Agent: Improved CLAUDE.md Bootstrap

**Changes:**
- Moved critical instructions to top of file (first thing Claude reads)
- Added explicit "your work is verified by automated programmatic tests"
- Added loop detection reminder in CLAUDE.md
- Added test/validation scripts discovery section
- Added GPU detection (nvidia-smi) for ML tasks
- Added CPU core count (nproc) for parallelism decisions
- Increased task description limit from 200 to 300 lines
- Removed "##" markdown formatting around task description (cleaner for Claude to parse)
- Increased build config display from 100 to 120 lines

### 6. Feature Prompt: Verification-Centric

**Changes:**
- Added "automated programmatic verifier" framing
- Added step 6 (verify again) to the workflow
- Added "Prefer simple, direct approaches" guidance
- Added loop detection self-check reminder
- Strengthened "every constraint, path, and format matters" language

## Strategy Analysis

### Why These Changes Should Improve Scores

1. **Effort high vs xhigh**: This is the #1 expected improvement. LangChain measured
   a 10+ point gap between xhigh-timeout and high-completion. Opus 4.7 at "high" is
   still extremely capable — avoiding timeouts means more tasks get committed solutions.

2. **Self-verification emphasis**: The prompt now strongly signals that verification
   is the most important step, compensating for the lower per-turn reasoning depth.

3. **Loop detection**: Prevents the agent from wasting 5+ minutes editing the same
   file with minor tweaks. Forces fundamental approach changes earlier.

4. **Pre-installed packages**: Eliminates 24.1% failure category (missing tools).
   Installing numpy/scipy/pandas/etc saves 30-60s per task that needs them.

5. **Better time management**: The relaxed deadlines (budget/6, 65% freeze, 85% final)
   give complex tasks more time while simple tasks still complete fast.

### General Patterns (Not Task-Specific)

All improvements are general-purpose:
- Effort level selection is agent-level configuration
- Self-verification prompting applies to any task
- Loop detection applies to any iterative coding task
- Pre-installed packages are common across task domains
- Time management tuning is domain-independent

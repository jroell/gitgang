## Changes Applied (This Run)

### 1. Retry-on-Early-Exit Mechanism (Major — Execution Error Recovery)

**Research basis:** Terminal-Bench 2.0's failure taxonomy shows ~50% of failures are
execution errors. Analysis of agent traces reveals that many failures happen early
in the time budget — the agent misunderstands the task, hits an unexpected error,
or takes a wrong approach. Top-scoring agents (Meta-Harness, ForgeCode) implement
retry mechanisms where a failed attempt informs the next one.

**Implementation:**
- Harbor agent now tracks elapsed time for each gitgang run
- If the agent exits before using 40% of its time budget, it retries with context:
  "A previous attempt FAILED. Here is what went wrong: [last 500 chars of output].
  Take a COMPLETELY DIFFERENT approach."
- Maximum 1 retry to avoid wasting the entire budget on retries
- Remaining time budget is recalculated (55% of original) for the retry attempt

**Expected impact:** +3-5% — converts early failures into second chances with
diagnostic context. The agent can learn from its first mistake and try differently.

### 2. Test Script Source Code in CLAUDE.md Bootstrap (Major — Verification Error Prevention)

**Research basis:** LangChain's harness engineering and ForgeCode's architecture both
emphasize "test scripts are ground truth." The current bootstrap lists test file
names but doesn't include their contents. This means the agent spends 2-5 turns
reading test scripts at runtime. Including them in CLAUDE.md means the agent
starts with full knowledge of what "correct" looks like.

**Implementation:**
- Bootstrap now finds test/validation scripts (test_*, *_test.*, verify*, check*,
  validate*, grade*, score*) and includes their first 150 lines in CLAUDE.md
- Makefile test targets are also included
- CLAUDE.md header now explicitly states: "The test/validation scripts below are
  GROUND TRUTH. Read them carefully — their assertions define correct behavior."

**Expected impact:** +2-4% — saves 2-5 exploration turns and ensures the agent
never misinterprets what the verifier checks. Agent starts with complete knowledge
of expected behavior.

### 3. Byte-Level Output Verification Instructions (Medium — Verification Errors)

**Research basis:** Terminal-Bench ICLR 2026 paper shows verification errors account
for ~25% of failures. Common patterns: trailing newlines, wrong encoding, numeric
precision mismatches, stdout vs stderr confusion.

**Implementation:**
- Added Rule 8: "When tests fail, compare ACTUAL vs EXPECTED output byte-for-byte.
  Use diff, xxd, or python3 repr() to see hidden characters."
- Added Rule 9: "If a task requires a server/daemon, test it end-to-end before committing."
- Step 2 (Analyze): Added checks for hidden requirements in test assertions
  (return codes, stderr vs stdout, file permissions)
- Error Recovery: Added specific patterns for encoding issues, numeric precision,
  and test format matching
- Before Finishing: Added explicit format verification step (wc -c, encoding check)
  and a "verify from scratch" step that re-runs tests in a clean subshell

**Expected impact:** +1-3% — directly addresses the verification error category.
Agents that verify byte-by-byte catch trailing newline issues, encoding problems,
and precision mismatches before final commit.

## Strategy Analysis

### Why These Changes Should Improve Scores

1. **Retry mechanism (+3-5%)**: Most impactful structural change. Converts early
   failures (misunderstood tasks, wrong initial approach) into second chances with
   diagnostic context. The agent gets to learn from its own failure.

2. **Test script in CLAUDE.md (+2-4%)**: Saves turns and prevents misinterpretation.
   The agent starts with complete knowledge of what the verifier checks, rather than
   discovering it mid-task.

3. **Byte-level verification (+1-3%)**: Directly targets verification errors (25%
   of all failures). Hidden characters, encoding, and precision are common gotchas.

4. **Model updates (+1-2%)**: GPT-5.5 is measurably better than 5.4 for coding.

5. **Ground truth emphasis (+0.5-1%)**: Reduces coherence errors where agent
   interprets task differently from verifier.

### Cumulative Expected Improvement: +7-15% over baseline

### General Patterns (Not Task-Specific)

All improvements are general-purpose patterns:
- Retry-on-early-exit applies to any agent that fails early
- Test script pre-reading applies to any task with validation scripts
- Byte-level verification applies to any output-checked task
- Model updates apply to all tasks
- Ground truth emphasis applies to any ambiguous specification

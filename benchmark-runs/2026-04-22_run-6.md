## Changes Applied (This Run)

### 1. Environment Bootstrapping (Major — inspired by Meta-Harness 76.4%)

Added pre-agent environment snapshot to the harbor agent. Before gitgang runs, the
agent now gathers a full snapshot of the sandbox environment and writes it to CLAUDE.md:
- Working directory path
- File listing (first 80 files, 3 levels deep)
- Available tools and their versions (python3, node, gcc, rustc, go, etc.)
- Build config contents (Makefile, package.json, requirements.txt, etc.)
- Task description (TASK.md, README.md, etc.)
- System memory info

This saves 2-5 exploration turns that the agent would otherwise spend on `find`, `which`,
`cat README.md`, etc. Claude Code auto-reads CLAUDE.md on startup, so the agent begins
with full context already in its prompt.

This is the primary technique used by Meta-Harness (Stanford IRIS Lab, 76.4% on tbench 2.0).

### 2. Claude Code --effort xhigh Flag

Added `--effort xhigh` to both the coder and reviewer Claude invocations. This is the
recommended default for Claude Opus 4.7 per Anthropic's best practices — it provides
deep reasoning without the full token cost of `max`, and is specifically optimized for
agentic coding and multi-step reasoning tasks.

### 3. Claude Code --bare Mode for Benchmarks

When running under a time budget (benchmark mode), Claude Code now starts with `--bare`
flag, which skips hooks, LSP, plugin sync, attribution, auto-memory, and background
prefetches. This reduces startup overhead significantly. CLAUDE.md context is still
provided via `--add-dir`.

### 4. System Prompt: Condensed and Literal for Opus 4.7

Complete rewrite of systemConstraints() based on Claude Opus 4.7 best practices:

**Key insight**: Opus 4.7 interprets instructions more literally than 4.6. The previous
prompt was verbose with nested bullet points — the new prompt is flat, direct, and
action-oriented.

Changes:
- Removed all category-specific planning guidance (Opus 4.7 doesn't need it — it does
  this naturally with xhigh reasoning)
- Replaced nested "### Step N" headers with flat "## Step N" structure
- Condensed error recovery from multi-level escalation to 3 lines
- Combined "Common fixes" into a single dense line instead of 7 bullet points
- Reduced prompt size by ~40% while retaining all critical guidance
- Changed "Observe → Implement → Commit → Verify → Iterate" to
  "Read → Code → Commit → Test → Fix → Commit" (more literal)

**Time management tightened:**
- First commit deadline: budget/8 (was budget/6) — ~105s for 840s budget
- Feature freeze: 60% of budget (was 65%) — earlier cutoff for stabilization
- Final commit: 80% of budget (was 85%) — more buffer before hard kill
- Command timeout: 30s (was 45s) — faster fail-and-pivot

### 5. Feature Prompt: Minimal for Opus 4.7

Reduced from 20 lines to 12 lines. Opus 4.7 doesn't need verbose repetition — it follows
literal instructions better with less noise in the prompt.

### 6. Harbor Agent: Split run() into Bootstrap + Pre-flight + Run

The previously monolithic run() command is now three separate exec_as_agent calls:
1. Environment bootstrap (CLAUDE.md generation + git init)
2. Pre-flight check (claude auth verification)
3. gitgang execution

This gives better error isolation — if bootstrapping fails, we see it clearly in
harbor logs rather than having it buried in the gitgang output.

## Strategy Analysis

### What top agents do differently (research findings)

1. **Environment bootstrapping** (Meta-Harness, 76.4%): Pre-gather env snapshot,
   inject into initial prompt. Saves 2-5 turns. ← **Implemented this run.**

2. **Native tool calling** (Terminus-KIRA, 74.7%): Replace ICL response parsing
   with native LLM tool definitions. Claude Code already does this natively.

3. **Marker-based polling** (Terminus-KIRA): Detect command completion early via
   echo markers. Claude Code handles this internally.

4. **Adaptive effort** (Opus 4.7): xhigh reasoning for complex tasks, lower for
   simple ones. ← **Implemented via --effort xhigh flag.**

5. **Scaffolding quality**: 2-6 point improvement from scaffold alone, independent
   of model. This is our primary optimization lever.

### Why these changes should improve scores

- **Env bootstrapping**: Eliminates the #1 time waste — environment exploration.
  Agent starts coding immediately instead of spending 2-5 turns on discovery.
- **--bare mode**: Reduces Claude Code startup from ~5-10s to ~1-2s. In a 840s
  budget, this matters.
- **--effort xhigh**: Calibrated reasoning depth. Not too shallow (high), not too
  expensive (max). Sweet spot for coding tasks.
- **Shorter prompt**: Less token overhead per turn. With 200 max turns, prompt
  savings compound.
- **Earlier first commit (budget/8 = ~105s)**: More aggressive than before. For
  simple tasks (file manipulation, scripts), 105s is plenty. For complex tasks,
  the agent can still iterate after the first commit.

## General Patterns (not task-specific)

All improvements are general-purpose:
- Environment bootstrapping applies to ANY task environment
- --bare and --effort flags are agent-level configuration
- Prompt condensation helps across all task types
- Earlier commit deadlines help all time-budgeted tasks

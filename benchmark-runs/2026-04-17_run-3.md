# Terminal-Bench 2.0 - Run 3

**Date:** 2026-04-17  
**Agent:** gitgang v1.9.5 (solo claude, claude-opus-4-6)  
**Dataset:** terminal-bench@2.0  
**Result:** Prompt engineering and Harbor agent improvements based on top-agent analysis

## Changes from Run 2

### Analysis of Top Agents
Studied Apex2 (64.5% with Sonnet 4.5), Terminus-2, Forge Code, and other high-scoring agents on the Terminal-Bench 2.0 leaderboard. Key findings:

- Top agents invest in targeted (not exhaustive) environment observation
- Task categorization with category-specific strategies is a consistent differentiator
- Commit-early-commit-often is critical for time-budgeted environments
- File writing discipline (heredoc escaping, read-back verification) prevents common failures
- Pre-installing build tools avoids mid-task dependency resolution delays
- Error recovery should escalate to fundamentally different approaches, not repeat
### Improvements Applied

#### 1. Targeted Observation Phase (systemConstraints)
- Changed from generic "spend 30-60s exploring" to a specific combined command that gathers context in a single shell call
- Added explicit checks for installed tools and packages
- Reduced exploration budget from 20% to 15% of total time
- Added "Do NOT exhaustively explore. Get what you need and move on."

#### 2. Category-Aware Planning (systemConstraints)
Expanded task categorization from 5 to 8 categories with more specific guidance:
- Algorithm/data structure tasks: identify core algorithm, test with provided examples first
- ML/training tasks: tiny test run first, check GPU availability
- Security/crypto tasks: verify tool versions, ground approach in exact commands
- System/DevOps tasks: inspect state before modifying, back up configs
- Optimization tasks: correct baseline first, measure before optimizing
- Debugging/fix tasks: reproduce bug first, read error messages carefully
- Build/compilation tasks: check compiler versions and flags
- Data processing tasks: check input format/encoding, verify output format matches spec exactly

Added "Identify potential failure modes upfront and have a backup approach ready."

#### 3. Improved Implementation Guidance
- Explicit heredoc best practices: use single-quoted `<<'EOF'` to prevent variable expansion
- Added file write verification: "After writing any file, verify it by reading it back"
- Added pip fallback patterns: `pip install --user X` or `python3 -m pip install X`
- Added apt-get recovery: `apt-get install -y --fix-broken`
- Added Python version check: `python3` vs `python`
#### 4. Better Verification Phase
- Added character-by-character output comparison guidance
- Added return code checking: `echo $?`
- Added numeric precision/formatting awareness
- Changed from "do not stop at first error" to "read the failure output, fix the issue, re-run"

#### 5. Expanded Error Recovery
- Common missing packages list: build-essential, python3-pip, curl, wget, unzip, cmake, pkg-config
- File debugging with `cat -A` for hidden characters
- Missing header -> dev package pattern
- Python import debugging checklist
- Hung command handling guidance
- Explicit "think about completely different solution path" when stuck

#### 6. Improved Time Management
- Reduced late-phase cutoff from 80% to 75% of budget
- Added explicit guidance to commit partial output if long-running commands are still going near deadline
- Tightened exploration budget from 20% to 15%

#### 7. Feature Prompt Improvements
- Added "Do not ask questions. Just execute."
- Added "Every word matters - pay attention to exact output formats, filenames, and constraints."
- Added explicit "Do not repeat failing strategies more than twice."
- Added "After committing, keep improving until time runs out."
- Added final verification reminder before declaring done
#### 8. Harbor Agent Improvements
- Upgraded from Node 20 to Node 22 LTS
- Pre-install common build tools: build-essential, cmake, pkg-config, python3-pip/venv/dev, libssl-dev, libffi-dev, etc.
- Explicitly set GITGANG_MAX_TURNS=200 in env (was only implicit from GITGANG_TIME_BUDGET_SECONDS)
- Updated default model reference from claude-sonnet-4-6 to claude-opus-4-6 in usage docs

### Research Basis
These improvements are based on analysis of:
- Apex2 (64.5%): Predict-Explore-Synthesize-Execute pipeline; targeted observation; risk-aware category prompting; heredoc management
- Terminus-2: Session recovery, long-running task management
- General leaderboard patterns: scaffolding contributes 2-6% on top of raw model capability; top agents pre-install dependencies; commit discipline is critical

## Changes from Run 1

### Previous Failure Mode (Run 1)
- AgentTimeoutError: Claude spent entire 900s budget exploring/thinking without committing
- Root cause: No structured execution workflow, no environment observation phase

### Improvements Applied (v1.7.0)

#### 1. Structured Execution Workflow in System Prompt
Replaced flat instruction list with a phased workflow: **Observe → Plan → Implement → Verify → Commit**

- **Observe phase**: Agent now explicitly told to spend 30-60s understanding the environment (list files, check available tools/languages, read task descriptions) before writing any code
- **Plan phase**: Task-type-aware strategies — different approaches for ML tasks (small test runs first), security tasks (ground commands before execution), optimization tasks (correct baseline first), etc.
- **Implement phase**: Emphasis on writing complete files, committing early and often, installing dependencies promptly
- **Verify phase**: Run tests/validation before declaring done, check for common errors
- **Commit phase**: Ensure all work is captured in git

#### 2. Time Budget Improvements
- Reduced exploration budget from "half" to "20%" — forces earlier code writing
- Added explicit late-phase guidance: after 80% of budget, stop adding features and focus on committing
- Added `--max-turns 200` for Claude when time-budgeted to prevent runaway loops

#### 3. Feature Prompt Improvements
- Reordered objectives: correctness first, then commit, then verification
- Added explicit instruction to observe workspace before modifying anything
- Added guidance to try different approaches when stuck (not repeat failing strategies)
- Emphasized writing complete files over incremental patches

#### 4. Harbor Agent Improvements
- Added `DISABLE_PROMPT=1` env var to prevent interactive prompts in benchmark
- Added `GITGANG_MAX_TURNS` support for explicit turn limits

#### 5. Error Recovery Guidance
- Explicit strategies: install missing tools, try different approaches, verify file writes by reading back
- Guidance on heredoc file writing best practices

### Research Basis
These improvements are based on analysis of high-performing Terminal-Bench 2.0 agents:
- **Apex2** (64.5%): Predict → Explore → Synthesize → Execute pipeline; environment observation; risk-aware category prompting
- **Terminus-2**: Single-tool tmux approach, session recovery, long-running task management
- **General leaderboard patterns**: Top agents invest in upfront observation, commit early, and validate before completion

### Scoring Note
This run focused on prompt engineering improvements that cannot be benchmarked without Docker/Harbor infrastructure. The improvements target the general failure modes identified in Run 1 and from studying the leaderboard:
1. Timeout from insufficient time awareness → improved time budget management
2. No environment observation → added Observe phase
3. No task categorization → added Plan phase with category-specific strategies
4. No validation → added Verify phase
5. Runaway agent loops → added --max-turns cap
